# mall-miniapp 跨端兼容改造说明

## 改造目标
使 uni-app 项目同时兼容微信小程序和 H5 页面，并且两个平台都使用 Vant UI 组件。

## 已完成的工作

### 1. 安装依赖
- 安装了 `vant@^4.9.1`（Vue 3 版本）用于 H5 平台
- 项目已包含 `@vant/weapp@^1.11.7` 用于微信小程序平台

### 2. 修改配置文件
**package.json**
- 修改了脚本命令，明确指定平台：
  - `dev:mp-weixin`: 微信小程序开发
  - `build:mp-weixin`: 微信小程序打包
  - `dev:h5`: H5 开发
  - `build:h5`: H5 打包

**vite.config.ts**
- 已配置 SCSS 变量
- 已配置开发服务器代理

**manifest.json**
- 已配置 H5 相关设置（publicPath、router 等）
- 已配置微信小程序 appid

### 3. 创建跨端兼容封装
**src/utils/vant.ts**
- 封装了 Vant API，同时支持 H5 和微信小程序
- 导出的 API：
  - `showToast`
  - `showDialog`
  - `showNotify`
  - `showLoadingToast`
  - `showSuccessToast`
  - `showFailToast`
  - `showImagePreview`
  - `closeToast`
  - `closeDialog`

### 4. 修改入口文件
**src/main.ts**
- 使用条件编译，在 H5/APP 平台引入并注册 Vant
- 在微信小程序平台，Vant 组件通过 `wxcomponents` 目录自动引入

### 5. 修改示例页面
**src/pages/vant-demo/index.vue**
- 修改了 import 语句，使用封装后的 Vant API（`@/utils/vant`）
- 在模板中添加了微信小程序端所需的组件（`<van-toast />`、`<van-dialog />`）

## 使用说明

### 开发微信小程序
```bash
npm run dev:mp-weixin
```
- 使用微信开发者工具导入 `dist/dev/mp-weixin` 目录
- Vant 组件通过 `wxcomponents/vant` 自动引入

### 开发 H5
```bash
npm run dev:h5
```
- 访问 `http://localhost:3003`（或控制台显示的地址）
- Vant 组件通过 `main.ts` 全局注册

### 打包微信小程序
```bash
npm run build:mp-weixin
```
- 打包后的文件在 `dist/build/mp-weixin` 目录

### 打包 H5
```bash
npm run build:h5
```
- 打包后的文件在 `dist/build/h5` 目录

## 在页面中使用 Vant 组件

### 使用 Vant UI 组件（标签方式）
在 template 中直接使用 Vant 组件标签，双端自动适配：
```vue
<template>
  <view>
    <van-button type="primary">主要按钮</van-button>
    <van-field v-model="value" label="用户名" placeholder="请输入" />
  </view>
</template>
```

### 使用 Vant API（函数方式）
在 script 中引入封装后的 API：
```vue
<script setup lang="ts">
import { showToast, showDialog } from '@/utils/vant'

const handleClick = () => {
  showToast('提示内容')
}
</script>
```

## 注意事项

### 1. 微信小程序端使用 Toast/Dialog 等 API
需要在页面模板中添加对应的组件：
```vue
<template>
  <view>
    <!-- 页面内容 -->
    
    <!-- 小程序端需要添加的组件 -->
    <!-- #ifdef MP-WEIXIN -->
    <van-toast id="van-toast" />
    <van-dialog id="van-dialog" />
    <van-notify id="van-notify" />
    <!-- #endif -->
  </view>
</template>
```

### 2. 组件属性差异
大部分 Vant 组件在双端的属性/事件是一致的，但部分组件可能有差异：
- 建议查阅 [Vant 官方文档](https://vant-contrib.gitee.io/vant/#/zh-CN/) 和 [@vant/weapp 文档](https://vant-contrib.gitee.io/vant-weapp/#/)
- 如有差异，使用条件编译分别处理

### 3. 样式差异
- 微信小程序使用 `wxcomponents/vant` 的样式
- H5 使用 `vant/lib/index.css` 的样式
- 自定义样式时，建议优先使用 uni-app 的 `rpx` 单位，自动适配双端

## 常见问题

### Q: H5 端 Vant 组件样式不正常？
A: 检查 `main.ts` 中是否正确引入了 Vant 样式：`import 'vant/lib/index.css'`

### Q: 微信小程序端 Vant 组件不显示？
A: 检查 `wxcomponents/vant` 目录是否存在，并且组件名称是否正确

### Q: API 调用报错？
A: 确保使用了封装后的 API（`@/utils/vant`），而不是直接从 'vant' 引入

## 下一步优化建议
